# 命名與連結規範

## 檔案命名
- 主題筆記檔名 = 主題名稱，例：`牛頓運動定律.md`，放在對應科目目錄下。
- 主題名稱用通用課綱名稱，不加冊別、不加編號（冊別寫在 frontmatter）。
- 同一概念只開一個檔，避免重複；相關子題用「## 區塊」或另開檔並互相連結。

## 跨科 / 跨主題連結
- 一律用**純主題名**：`[[主題]]`，例：`[[向量]]`、`[[動量守恆]]`。跨科、同科都一樣（檔名＝主題，Obsidian 以 basename 跨資料夾自動解析）。
- ⚠️ **不要加「科目-」前綴**（如 `[[物理-向量]]`）。檔名不含科目，這種寫法會解析失敗、在關聯圖上斷鏈。
- 若不同科出現**同名主題**（罕見），改用路徑明確指定：`[[科目/主題]]`，例：`[[物理/週期]]` vs `[[化學/週期]]`。
- 連結要有意義：只在「讀 A 時真的需要懂 B」時才連，避免連結爆炸。

## 標籤（frontmatter `標籤`）
- 用主題群組，例：`[力學, 運動學]`、`[氧化還原, 電化學]`。
- 標籤是橫向分類，連結是縱向關聯，兩者並用。

## 科目目錄對照
國文 / 英文 / 數學 / 物理 / 化學 / 生物 / 地球科學 / 歷史 / 地理 / 公民與社會

## 每科 MOC（主題地圖）
- 每個科目目錄放一個 `<科目>MOC.md`，列出該科所有主題並分章節組織，作為該科入口。
- 全庫總入口為 `_index/主題地圖.md`。

## 關聯圖（Graph view）約定
關聯圖的定位是**學習路線圖**：呈現「已建筆記 ＋ 各科 MOC 課程骨架 ＋ 尚未建立的主題」，讓孩子一眼看出主題如何關聯、下一步可往哪裡加廣加深。

- 專案內建關聯圖預設篩選（`.obsidian/graph.json`，隨版控發佈），顯示範圍：
  - **各科主題筆記**（已建）。
  - **各科 `<科目>MOC.md` 與 `_index/主題地圖.md`**：作為課程地圖的中樞節點（hub），把整科主題串起來。
  - **未建立的主題節點**（`hideUnresolved: false`）：MOC 與筆記裡 `[[尚未建檔主題]]` 會以「幽靈節點」顯示，這正是引導學習廣度／深度的路標。
- 隱藏範圍（靠 `search` 的 `-path:` 排除）：`_system`、`_review`、`docs/`、`README`、`CLAUDE`、`CONTRIBUTING`、`LICENSE`、`免責聲明`。`.obsidian`、`.claude` 為點開頭資料夾，Obsidian 自動忽略。
- 排除採**明確列舉**而非「凡含底線即隱藏」；因此主題筆記檔名含底線雖不建議（易與系統檔混淆），但已不會導致它從關聯圖消失。
- **顏色導航約定**（`colorGroups`）：全庫總圖 `主題地圖` 標**金色**、各科 MOC（檔名 `<科目>MOC.md`，例 `數學MOC`）標**橘色**，作為可點擊進入的「中樞節點」；未建主題維持暗淡（幽靈節點），定位為視覺路標。各科 MOC 節點在關聯圖上會顯示科目名稱（如「數學MOC」），不再是難以分辨的「_MOC」。
  - ⚠️ Obsidian 原生**無法讓「點擊幽靈節點」跳到 MOC**（點幽靈節點只會建立空白檔）。因此導航設計是：孩子看到想學的暗淡幽靈主題 → 改點它**相鄰的橘色該科 MOC**（或金色總圖）→ 進入該科主題地圖，裡面有完整待學清單與 `/learn <科目> <主題>` 生成指令。亮節點＝可操作入口，幽靈節點＝「還有這些可以學」的路標。
- 此設定為**共用預設**；個人若想改成另一種檢視（例如關掉幽靈節點、只看已建筆記），在 Obsidian UI 內調整即可（不必改檔），你的個人工作區狀態（`workspace.json`）不會進版控。
