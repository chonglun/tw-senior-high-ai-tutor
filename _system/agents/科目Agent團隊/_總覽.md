# 各科 Agent Team 總覽

> **設計檔 vs 執行檔**：本目錄（`_system/agents/`）是**人類可讀的設計定義**；可在對話中呼叫的**執行版 subagent** 在 `.claude/agents/`（10 隻 tutor-* + note-writer/checker/linker），由 `/learn <科目> <主題>` 指令串成產線。tutor-* 的 system prompt 直接內嵌本目錄各科定義，兩者一致。修改教學法時兩邊都要更新。

10 科 × 每科最多 3 隻 = 共 30 隻教學法原型 Agent。全部去識別化（研究參考真實名師，最終命名不含姓名）。每隊依固定協作節奏：**引發興趣 → 加速理解 → 強化記憶/應試**。

## 共通協作原則
1. **接力而非並列**：依「興趣→理解→記憶」順序，後者引用前者產物（如數學軍師引用建築師的速記公式）。
2. **互相把關**：負責趣味/比喻的 Agent 產物，須由負責嚴謹的 Agent 檢查（物理鐵則：比喻用完要說「哪裡會破功」）。
3. **產物交棒**：團隊整合出的草稿 → 交 [筆記轉換Agent](../筆記轉換Agent.md) 入庫 → [內容檢查Agent](../內容檢查Agent.md) 把關。
4. **去識別化紀律**：見 [名師風格Agent總則](../名師風格Agent總則.md)，不得使用真實教師姓名。
