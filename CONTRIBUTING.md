# 🤝 貢獻指南（Contributing）

謝謝你想讓這個專案更好！本專案的核心價值是「**好懂好記 + 誠實正確**」，所以貢獻規範也圍繞這兩點。請先讀 [README](README.md) 與 [免責聲明](免責聲明.md) 了解專案定位。

> 一句話：**內容寧可保守、不可杜撰；不確定就標出來交人工複核，絕不假裝全對。**

---

## 你可以貢獻什麼

- 📝 **新主題筆記**：用 `/learn <科目> <主題>` 產生，補齊各科 `<科目>MOC.md` 裡還是 `[ ]` 的主題。
- 🔍 **訂正既有筆記**：修正錯誤的公式、年代、課綱範圍、錯別字，或補強易錯點與自我檢測。
- 🧠 **改進教學法 Agent**：讓某科講得更清楚、比喻更貼切（改 `_system/agents/` 設計，並同步 `.claude/agents/`）。
- 🛠 **改進系統**：模板、規範、產線指令、文件。

---

## 開始之前

1. Fork 並 clone 本專案。
2. 安裝 Claude Code（見 [快速開始](docs/快速開始.md)），這是跑產線的必要工具。
3. 安裝 pre-commit hook（**一次性**，確保提交前品質自動把關）：
   ```
   sh scripts/install-hooks.sh
   ```
4. 建議用 [Obsidian](https://obsidian.md) 開啟資料夾，方便預覽 `[[連結]]`、Mermaid 與關係圖譜。
5. 從 `main` 開一個分支：`git checkout -b note/<科目>-<主題>` 或 `fix/<說明>`。

## 新增 / 修改筆記的流程

**強烈建議透過產線產生，而非手寫整篇**，以確保品質一致：

```
/learn <科目> <主題>
```

它會自動跑完：科目教學 Agent → `note-writer` 入庫 → `note-checker` 查證標信心度 → `note-linker` 建跨科連結。

若你是**手動修改**既有筆記裡的事實內容（公式、定義、年代、字根字義…），**改完務必再跑一次內容檢查**重新查證你新加的硬事實，不要留下未驗證的主張：

```
請用 note-checker 校驗 <科目>/<主題>.md
```

### 提交前自我檢查（對照 `_system/規範/內容檢查清單.md`）

- [ ] 硬事實（公式/年代/數據/課綱範圍）已查證，來源可靠（教科書、大考中心）。
- [ ] 比喻與生活連結**科學正確**，不建立錯誤直覺；「為什麼要學」的好處真實、不誇大。
- [ ] 模板區塊齊全且順序正確（💡 為什麼要學在開頭）；自我檢測 ≥ 3 題且答案與內文一致。
- [ ] 不確定處已標 `（待查）` 或寫進「待確認項」，**未杜撰**。
- [ ] frontmatter 的 `檢查狀態` 與 `信心度` 已由 `note-checker` 標註（不要自填「高」蓋過檢查）。
- [ ] 逐字校對：無錯別字、形近/音近誤用、亂碼、全形半形或單位符號問題。

## 命名與連結規範（節錄 `_system/規範/命名與連結規範.md`）

- 檔名 = 主題名稱，用通用課綱名稱，不加冊別/編號（冊別放 frontmatter）；同一概念只開一個檔。
- 跨科連結一律用純主題名 `[[主題]]`（例：`[[向量]]`）；同名主題撞名才用 `[[科目/主題]]`。**只在「讀 A 真的需要懂 B」時才連**。
- 新增筆記後，記得更新該科 `<科目>MOC.md` 與 `_index/主題地圖.md`（`note-linker` 會幫忙，但請確認）。
- 一律使用**繁體中文（台灣用語）**。

## 紅線（會被退回的 PR）

- ❌ **杜撰或未查證的事實**，或把信心度灌水成「高」。
- ❌ **逐字複製受著作權保護的教材、他人筆記，或模仿任何實名教師**（人設、姓名、可辨識標誌）。本專案只萃取可公開的教學法原型，見 [`名師風格Agent總則`](_system/agents/名師風格Agent總則.md)。
- ❌ 為了有趣而誤導的比喻；寧可樸素，不可錯誤。
- ❌ 夾帶個資、密鑰，或把 `settings.local.json`、`.DS_Store` 等加入版控。

---

## 送出 Pull Request

1. 一個 PR 聚焦一件事（一篇筆記 / 一處修正 / 一個 Agent 改進），方便審查。
2. PR 說明請寫：**改了什麼、為什麼、硬事實的查證來源**；若仍有「待確認項」請列出。
3. 維護者會對照內容檢查清單與課綱對齊審查；可能請你補來源或調整措辭。
