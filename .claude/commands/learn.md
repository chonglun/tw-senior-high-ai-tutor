孩子要學的內容是：**$ARGUMENTS**

請依序執行學習產線（用 Task 工具呼叫對應 subagent，前一步的輸出餵給下一步）：

## 1. 生成教學內容
解析出「科目」與「主題」。依科目對照表呼叫對應的科目教學 subagent，把「主題」交給它，取得教學草稿：

| 科目 | subagent |
|---|---|
| 國文 | tutor-chinese |
| 英文 | tutor-english |
| 數學 | tutor-math |
| 物理 | tutor-physics |
| 化學 | tutor-chemistry |
| 生物 | tutor-biology |
| 地球科學 | tutor-earth |
| 歷史 | tutor-history |
| 地理 | tutor-geography |
| 公民與社會 | tutor-civics |

若科目不在表中，告訴孩子目前支援哪些科目並停止。

## 2. 入庫
呼叫 `note-writer`，把「科目、主題、教學草稿」交給它，依模板寫入 `<科目>/<主題>.md`。若它回報檔案已存在，向孩子確認是否覆蓋，不要擅自覆蓋。

## 3. 檢查
呼叫 `note-checker` 校驗剛入庫的檔，讓它查證硬事實、更新檢查狀態/信心度與待確認項。

## 4. 串連
呼叫 `note-linker` 為該筆記建立跨科連結並更新主題地圖與該科 MOC。

## 5. 回報孩子
用溫暖鼓勵的中文回報：
- ✅ 完成的檔案路徑（可點開複習）
- 📊 信心度，以及⚠️需要人工確認的點（誠實說明哪些還要老師/家長確認）
- 🎯 這個主題最該記住的一句話
- 一句鼓勵的話

提醒：整條產線重在「正確 + 好記」。若檢查 Agent 標了 `待人工複核`，務必如實告訴孩子，不要假裝一切都正確。
